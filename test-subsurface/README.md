# Test Subsurface Demo

这是一个基于 SDL3 与原生 Wayland 协议编写的复合表面（Subsurface）特性测试 Demo。用于测试 Wayland 合成器对各种子表面的渲染、排列、提交更新等特性的支持情况。

## 编译方法

在工程根目录（`treeland-demo-client`）执行 CMake 编译即可：

```bash
cmake -S . -B build
cmake --build build
```

## 运行方法

运行生成的可执行程序：

```bash
./build/test-subsurface/test-subsurface
```

## 测试内容与交互说明

启动 Demo 后，屏幕顶部会显示 5 个按钮块。可以使用鼠标点击来主动添加或移除子表面：

1. **Add Sync (红色框)**
   - **行为**: 添加一个默认的 **同步子表面 (Synchronized Subsurface)**。
   - **特性**: 对其进行的任何状态更改（如移动位置、附加新缓冲区）都将进入挂起（Pending）状态，直到其所属的父表面（主窗口）提交后才会统一生效。
   
2. **Add Desync (绿色框)**
   - **行为**: 添加一个 **异步子表面 (Desynchronized Subsurface)**。
   - **特性**: 它的提交状态与父表面解绑，当调用自身的 `wl_surface_commit` 时，它的状态更新会立刻被应用到屏幕上。

3. **Add Above (蓝色框)**
   - **行为**: 添加一个通过 `wl_subsurface_place_above` 指定在主表面 **上方** 的子表面。
   - **特性**: 覆盖在主窗口内容上。

4. **Add Below (黄色框)**
   - **行为**: 添加一个通过 `wl_subsurface_place_below` 指定在主表面 **下方** 的子表面。
   - **特性**: 渲染在主表面下层。为了能让此层可见，主窗口启用了 `SDL_WINDOW_TRANSPARENT`，并且在渲染背景时使用了带有 Alpha 通道的半透明颜色。

5. **Z-Toggle (青色框)**
   - **行为**: 添加一个会**周期性自动切换 Z 轴层级**的子表面。
   - **特性**: 它会在主表面“上方 (Above)”和“下方 (Below)”之间每秒自动交替切换，用于验证合成器对动态层级调整特性的支持。

6. **Remove (紫色框)**
   - **行为**: 删除最后一次创建的子表面（后进先出，LIFO）。

7. **Static Out (下一行的灰色框)**
   - **行为**: 在主表面的四周边缘外部（大概率超出主表面的可视范围）添加一个**静态的（不移动的）**子表面。
   - **特性**: 它的坐标 `x` 或 `y` 在初始化时就被随机指派在了主表面的边界或超出边界的地方，用于验证合成器对于越界渲染（剪裁）和超出父边界区域定位的处理。

*注：程序会自动给每个子表面加上简单的上下浮动动画，以便于清晰辨认它们是否能够独立渲染，以及检查它们是否能够正确响应同步/异步特性的更新限制。*
